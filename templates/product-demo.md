# Product Demo Template

A 12-slide structure for product demonstrations and feature presentations.

## Template Structure

Use this structure when creating product demos. Each section becomes a slide.

```
[Title]
[Product Name]
Product Demo
---
[The Problem]
What challenge are we solving?
- Pain points
- Current workarounds
- Cost of status quo
---
[The Solution]
Introducing [Product Name]
- Core value proposition
- Key differentiators
- Why it matters
---
[How It Works]
The key workflow
- Step 1: [Action]
- Step 2: [Action]
- Step 3: [Action]
---
[Feature 1]
[Feature Name]
- What it does
- How to use it
- Benefits
---
[Feature 2]
[Feature Name]
- What it does
- How to use it
- Benefits
---
[Feature 3]
[Feature Name]
- What it does
- How to use it
- Benefits
---
[Integration]
Works with your existing tools
- Integrations available
- API/SDK options
- Data portability
---
[Results]
What users achieve
- Key metrics
- Customer quotes
- Before/after
---
[Pricing]
Plans and options
- Tier overview
- Key features per tier
- Enterprise options
---
[Getting Started]
Start using [Product Name] today
- Signup process
- Onboarding
- Support resources
---
[Q&A]
Questions?
- Contact information
- Resources
- Next steps
```

## Usage Notes

### Tone
- Clear and educational
- Show, don't just tell
- Focus on user benefits

### Customization Points
- Replace [placeholders] with actual features
- Add screenshots or demo recordings
- Tailor to audience's use case

### Demo Flow Tips
- Start with the problem they feel
- Show the "aha moment" early
- Cover top 3-5 features, not everything
- End with clear next step

### Audience-Specific Variations
- **Technical audience**: More on architecture, APIs, security
- **Business audience**: More on ROI, time savings, case studies
- **End users**: More on daily workflow, ease of use

## Gamma Parameters

When using this template:
```json
{
  "textMode": "generate",
  "numCards": 12,
  "textOptions": {
    "tone": "clear, educational, exciting",
    "audience": "prospects, users, technical evaluators"
  }
}
```
